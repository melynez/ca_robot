# ca_robot

Cellular Automata Robots (CARs): a framework for generating, perturbing, classifying, and analyzing robot trajectories derived from Elementary Cellular Automata (ECA).

This repository contains the code used to:

- Generate CAR trajectories from ECA rules.
- Generate random-control trajectories using shuffle and white-noise bit sources.
- Generate barrier-interaction trajectories.
- Classify movement types using a trained Random Forest classifier.
- Analyze barrier reaction types.
- Compute bit-retention metrics (BRI).
- Evaluate CAR policies in MiniGrid environments.

---

# Overview

A Cellular Automata Robot (CAR) is generated by:

1. Evolving a 1D Elementary Cellular Automaton.
2. Converting each generation's bitstring into a 2D movement trajectory.
3. Tracking the resulting robot path over time.

The repository investigates whether emergent movement behaviors are specific to CA-generated bitstrings or can also arise from randomized controls.

Controls include:

- Shuffle controls
- White-noise controls

Experiments include:

- Original trajectories
- Barrier-interaction trajectories
- Movement-type classification
- Barrier-reaction classification
- Bit-retention analysis
- MiniGrid evaluation

# Pipeline

The typical workflow is:

```text
Generate CA trajectories
            ↓
Generate random controls
            ↓
Predict movement types
            ↓
Predict barrier movement types
            ↓
Classify barrier reactions
            ↓
Compute BRI metrics
            ↓
Run MiniGrid experiments
```

---

# 1. Generate CA Trajectories

Produces:

- Original CAR trajectories
- Barrier CAR trajectories
- Barrier metadata

Example:

```bash
python src/car_generation/generate_car_path.py ^
    --out_dir outputs/path_data ^
    --initial_state 0001000 ^
    --num_gen 100 ^
    --barrier_gen 49 ^
    --barrier_extend 20 ^
    --barrier_angle 90 ^
    --rules_start 0 ^
    --rules_end 255 ^
    --workers 1
```

Outputs:

```text
outputs/path_data/
└── 0001000_100/
    ├── original/
    └── gen49_size20_angle90_extra0/
```

---

# 2. Generate Shuffle and White-Noise Controls

Uses CA-generated trajectories as references.

Produces:

- Shuffle trajectories
- White-noise trajectories

Example:

```bash
python src/car_generation/generate_car_control_path.py ^
    --reference_root outputs/path_data ^
    --out_dir outputs/random_controls ^
    --initial_state 0001000 ^
    --num_gen 100 ^
    --barrier_gen 49 ^
    --barrier_extend 20 ^
    --barrier_angle 90 ^
    --barrier_modes easy
```

Outputs:

```text
outputs/random_controls/
└── 0001000_100/
```

---

# 3. Predict Original Movement Types

Uses the trained Random Forest classifier.

Example:

```bash
python src/movement_classification/predict_movement_og.py ^
    --data_root outputs/path_data ^
    --model_dir models ^
    --out_dir outputs/movement_original_ca
```

Outputs:

```text
full_predictions.csv
full_features_raw.csv
class_distribution.csv
```

Movement classes:

- Line
- Curve
- Hills
- Block
- Wobble
- Zigzag
- Meandering
- Noisy

---

# 4. Predict Barrier Movement Types

Classifies movement behavior before and after barrier interaction.

Example:

```bash
python src/movement_classification/predict_movement_barrier.py ^
    --data_root outputs/path_data ^
    --model_dir models ^
    --out_dir outputs/movement_barrier_ca ^
    --barrier_dir_name gen49_size20_angle90_extra0
```

Outputs include:

```text
barrier_predictions_per_segment.csv
barrier_change_summary.csv
barrier_transition_matrix.csv
```

---

# 5. Barrier Reaction Type Classification

Classifies barrier responses into:

- Extinction
- Loop
- Scrambled
- Trajectory Change
- Shift

Example:

```bash
python src/barrier_reaction_type/barrier_reaction_type.py ^
    --ca_original_dir outputs/path_data/0001000_100/original ^
    --ca_barrier_dir outputs/path_data/0001000_100/gen49_size20_angle90_extra0 ^
    --random_barrier_dir outputs/random_controls/0001000_100/gen49_size20_angle90_extra0 ^
    --out_dir outputs/barrier_reaction_analysis
```

# 6. Bit Retention Index (BRI)

```bash
python src/bit_retention/compute_bri_CA.py ^
    --regular_original_dir ... ^
    --regular_barrier_dir ... ^
    --out_dir outputs/bri
```

# 7. MiniGrid Experiments

Evaluates CAR-derived action policies in MiniGrid.

Example:

```bash
python src/minigrid/minigrid_exact_retry.py ^
    --out_dir outputs/minigrid ^
    --rules_start 0 ^
    --rules_end 255 ^
    --seeds 10 ^
    --sources ca shuffle white_noise
```
